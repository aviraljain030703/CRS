## Step 3: Configure Environment Variables

Before deployment, Render needs these environment variables:

| Variable | Value | Required |
|----------|-------|----------|
| `FLASK_ENV` | production | ✅ |
| `SECRET_KEY` | Generated by Render | ✅ |
| `JWT_SECRET_KEY` | Generated by Render | ✅ |
| `CMS_DATABASE_URL` | Auto-filled by Render | ✅ |
| `SEED_DB` | true (first run), false (after) | ✅ |
| `SESSION_COOKIE_SECURE` | true | ✅ |
| `CORS_ORIGINS` | https://yourdomain.com | ✅ |
| `MAIL_SERVER` | smtp.gmail.com | ⚠️ Optional |
| `MAIL_USERNAME` | your-email@gmail.com | ⚠️ Optional |
| `MAIL_PASSWORD` | app-specific-password | ⚠️ Optional |

## Step 4: Deploy with Blueprint (EASIEST)

### Using render.yaml (Recommended)

Render can auto-detect and deploy from `render.yaml`:

1. Go to https://render.com/dashboard
2. Click **New +** → **Blueprint**
3. Paste your GitHub repo URL: `https://github.com/YOUR_USERNAME/campus-grievance-hub`
4. Click **Connect**
5. Select branch: `main`
6. Click **Deploy**

✅ **Render auto-deploys:**
- Python 3.11 environment
- MySQL database
- Web service
- All environment variables

---

## Step 5: Manual Deployment (If Blueprint Doesn't Work)

### Create Web Service

1. Go to https://render.com/dashboard
2. Click **New +** → **Web Service**
3. **Connect to GitHub**:
   - Select your repository
   - Select branch: `main`
4. **Configure**:
   - **Name**: `campus-grievance-hub`
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r flask-app/requirements.txt`
   - **Start Command**: `cd flask-app && gunicorn --bind 0.0.0.0:$PORT --workers 4 wsgi:application`
   - **Plan**: Free (or Starter at $7/month)
5. Click **Advanced**:
   - **Root Directory**: `flask-app`
   - **Auto-deploy**: Turn ON
6. Click **Create Web Service**

### Add Environment Variables

After service is created:

1. Go to **Environment**
2. Add these variables:

```
FLASK_ENV = production
FLASK_DEBUG = false
SECRET_KEY = <auto-generated>
JWT_SECRET_KEY = <auto-generated>
SEED_DB = true
SESSION_COOKIE_SECURE = true
CORS_ORIGINS = https://yourdomain.com
PYTHON_VERSION = 3.11
```

### Create MySQL Database

1. Go to Dashboard
2. Click **New +** → **MySQL**
3. **Configure**:
   - **Name**: `cms-database`
   - **Database**: `cms_db`
   - **User**: `cms_user`
   - **Plan**: Free tier (or Starter)
4. Click **Create Database**
5. Copy the connection string

### Connect Database to Web Service

1. Go back to Web Service
2. Go to **Environment**
3. Add:
   ```
   CMS_DATABASE_URL = <paste MySQL connection string>
   ```
4. Click **Deploy**

## Step 6: Database Setup

### Initial Setup (First Deployment)

Set `SEED_DB=true` to auto-populate database:

```
SEED_DB = true
```

This will:
- Create all tables
- Seed sample data
- Create admin user

### After Initial Deployment

Change to prevent re-seeding:

```
SEED_DB = false
```

## Troubleshooting

### Service Won't Deploy

**Check logs:**
1. Dashboard → Logs tab
2. Look for error messages

**Common issues:**
- Missing `requirements.txt`
- Wrong start command
- Missing environment variables

**Solution:**
```bash
# Verify requirements.txt exists
ls flask-app/requirements.txt

# Verify Procfile or render.yaml
cat Procfile
cat render.yaml
```

### Database Connection Failed

**Check connection string:**
```
CMS_DATABASE_URL = mysql+pymysql://user:pass@host:3306/database
```

**Verify in Render:**
- Go to MySQL → Info
- Copy connection string
- Add to Web Service environment

### Application Crashes on Start

**Common cause:** Database not initialized

**Solution:**
1. Set `SEED_DB=true`
2. Restart service
3. Check logs
4. Set `SEED_DB=false` after initialization

## Performance Tips

### Optimize for Render

1. **Use Starter Plan** ($7/month) for better performance
2. **Increase Worker Processes**: Update start command
   ```
   gunicorn --workers 8 --worker-class sync wsgi:application
   ```
3. **Enable Caching**: Configure Redis (optional)
4. **Optimize Database**: Add indexes

### Monitor Performance

1. Dashboard → **Metrics** tab
2. Watch CPU, memory, disk usage
3. Scale if needed (Settings → Instance Type)
